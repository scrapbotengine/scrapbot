# ADR-002: Use text-first project files

**Date:** 2026-07-07

## Context

Scrapbot projects are intended to be ordinary directories that users can inspect, edit, version, and hand to agents. The first runtime slice needs a project manifest, a default scene, and a format that stays readable before a full editor exists.

## Decision

Use `project.toml` as the project manifest and TOML scene files under the project directory. The initial default scene is `scenes/main.scene.toml`.

## Consequences

Projects are easy to create, diff, and edit with normal tools. Agents can read and update project files without binary format tooling, and `scrapbot check` can validate projects without opening a window.

The current implementation intentionally supports only the TOML subset generated by Scrapbot. This keeps the first slice small, but the project must either adopt a real TOML parser or formalize the supported subset before scene files become richer.
