## Feature Development

- Check the feature list/roadmap in `README.md`. If the feature is not listed, or already implemented, ask the user for guidance.
- Check this project's FDRs for detailed information about the planned feature, or feature change.
- Make sure all feature additions and changes are thoroughly tested with unit tests, integration tests, and visual regression tests as appropriate.
- Make sure all documentation -- `README.md`, ADRs, FDRs, inline comments, documentation website -- are updated to reflect the new feature or feature change.
