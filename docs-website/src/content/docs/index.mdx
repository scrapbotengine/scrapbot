Scrapbot is a research engine. It is not trying to be stable yet. It is trying to make the engine, the project files, and the agent workflow line up cleanly enough that we can keep building without losing the thread.

## What exists today

<CardGrid>
	<Card title="Text-first projects" icon="document">
		Projects are ordinary directories with `project.toml`, TOML scene files, Luau scripts, generated editor types, and optional native extension source.
	</Card>
	<Card title="ECS runtime" icon="setting">
		The runtime builds an ECS world from scene data, assigns generation-aware entities, validates component schemas, and executes scheduled systems.
	</Card>
	<Card title="Luau scripting" icon="code">
		Projects define components and systems in `scripts/main.luau`, query entities by component handles, and get generated Luau types from `scrapbot check`.
	</Card>
	<Card title="Native extensions" icon="rocket">
		Projects can declare Odin native extension targets. Scrapbot builds source-stamped dynamic libraries and loads schemas before Luau runs.
	</Card>
	<Card title="WebGPU rendering" icon="laptop">
		The `wgpu` backend renders ECS cube renderables through SDL3 and `wgpu-native`; the null backend supports fast smoke tests.
	</Card>
	<Card title="Hot reload" icon="refresh">
		Runtime hot reload covers `project.toml`, the default scene, `scripts/main.luau`, native extension libraries, and declared native source.
	</Card>
</CardGrid>

## The loop

```sh
mise build
mise scrapbot -- init my-game "My Game"
mise scrapbot -- check my-game
mise scrapbot -- run my-game
```

For the checked-in example:

```sh
mise scrapbot -- check examples/minimal
mise scrapbot -- run examples/minimal
```
