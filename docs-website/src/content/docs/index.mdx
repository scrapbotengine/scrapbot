## Why Machina Exists

<CardGrid stagger>
	<Card title="Text-first projects" icon="document">
		Projects and scenes are TOML. Scripts are Luau. Native extensions are Zig. Core state is designed to be read, edited, diffed, tested, and live-reloaded.
	</Card>
	<Card title="Real ECS everywhere" icon="setting">
		Game state, script systems, native systems, render extraction, UI primitives, editor overlays, and internal render systems all use the same ECS runtime model.
	</Card>
	<Card title="Agent-friendly loops" icon="rocket">
		Headless commands validate projects, step simulations, run game-shaped tests, benchmark scenes, and render deterministic BMP artifacts for verification.
	</Card>
	<Card title="Prototype, then port" icon="code">
		Write gameplay in typed Luau systems first, then move hot paths to project-local Zig systems without changing scene data or component ids.
	</Card>
	<Card title="Rendering is data-driven" icon="star">
		Scene-authored transforms, geometry, materials, cameras, lights, shadows, batching, UI, and editor overlays all flow through ECS data.
	</Card>
</CardGrid>
