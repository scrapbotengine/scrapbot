---
title: Project Layout
description: Understand the files that make up a Scrapbot project directory.
---

A Scrapbot project is a normal directory. The engine treats text files as the source of truth so humans, tools, and agents can inspect and edit them directly.

```text
my-game/
├── project.toml
├── scenes/
│   └── main.scene.toml
├── scripts/
│   └── main.luau
├── types/
│   └── scrapbot.d.luau
├── native/
│   └── myextension/
│       └── myextension.odin
└── build/
    └── extensions/
        ├── .scrapbot-extensions
        └── myextension-<source-stamp>.dylib
```

`build/` is generated output. Do not treat it as source.

## `project.toml`

The manifest names the project, selects the default scene, and can declare native extension targets.

```toml
name = "Minimal Example"
default_scene = "scenes/main.scene.toml"

[[native_extensions]]
name = "scrappyphysics"
source = "native/scrappyphysics"
```

The parser currently supports Scrapbot's narrow TOML subset, not all TOML syntax.

## Scene files

Scene files describe entities, built-in components, and schema-backed custom component data.

```toml
[[entities]]
name = "Left Cube"

[entities.transform]
position = [-2.25, 0, 0]
rotation = [0, 0, 0]
scale = [1, 1, 1]

[entities.geometry]
resource = "cube"

[entities.material]
resource = "coral"

[entities.components.autorotate]
velocity = [0, 1.5707963, 0]
```

Built-in sections currently include:

- `[entities.transform]`
- `[entities.camera]`
- `[entities.geometry]` and `[entities.material]`

Custom components use `[entities.components.<name>]`. Single-token names such as `autorotate` are project-level components. Dotted names such as `scrappyphysics.rigidbody` are engine or library components and must be registered before validation succeeds.

## Scripts

`scripts/main.luau` is the current project entry point. It runs after the scene is loaded and before systems step.

Scripts usually do three things:

- register project component schemas;
- retrieve built-in or native component handles;
- register systems that operate over queries.

## Generated Luau types

`types/scrapbot.d.luau` is generated by `scrapbot check`. It describes the `scrapbot` global, built-in component handles, project component payloads, library component payloads, and readonly query payload aliases.

Run this after changing component schemas:

```sh
mise scrapbot -- check my-game
```

## Native source and build output

Native extension source belongs under a project-local source directory such as `native/scrappyphysics`. Extensions can register component schemas and scheduled native systems. Declared targets are built into source-stamped dynamic libraries under `build/extensions`, with `.scrapbot-extensions` recording the active outputs:

```sh
mise scrapbot -- build examples/minimal
```

`check` and `run` also build declared native extensions automatically. With `--hot-reload`, changes inside declared native source directories rebuild the extensions before the runtime reloads the scene and script.
